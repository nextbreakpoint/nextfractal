Description

NextFractal is an application for creating fractals and other graphics artifacts. NextFractal has a real-time, multi-threaded graphics engine with layers, filters, effects, a lot of formulas, and parameters. NextFractal supports operations such as continuous zoom of Mandelbrot and Julia sets, rotation and color shifting. NextFractal has also advanced features such as networked computation and scripting. You can take photos of your fractals or record movies from your exploration paths. Photos and movies are stored in a database to be rendered in high resolution and exported to a file. NextFractal can be extended by plug-ins which provide fractals, formulas, filters, effects and more.

How to install the application

Unzip the archive in your preferred location. NextFractal requires Java SE version 1.6 or later for Windows and Linux, and version 1.5 or later for Mac OS X. You can download latest Java SE for Linux and Windows from http://java.sun.com/javase/downloads, and for Mac from http://developer.apple.com/java/download. NextFractal has been tested successfully on Ubuntu 9.10, Debian GNU/Linux 5.0, Windows XP (service pack 3), Windows Vista (service pack 2), Mac OS X 10.5 (Leopard) and Mac OS X 10.6 (Snow Leopard). For any problems, contact the author.

How to run the application

Click on the application icon to run NextFractal. Alternatively, you can run the application from the command line with the command:

java -Xms512M -Xmx1024M -Dosgi.install.area=. -Dosgi.configuration.area=configuration/NextFractal -jar plugins/org.eclipse.equinox.launcher_<bundle-version>.jar -application com.nextbreakpoint.nextfractal.runtime.swing.application

How to configure networked computation

NextFractal supports networked computation based on the JXTA peer-to-peer technology. To enable networked computation, open the ports from 9701 to 9704 in your firewall and select the JXTASpool in jobs tab of service window of NextFractal. When you select the JXTA spool, NextFractal automatically discovers other instances of NextFractal and of JXTASpool service on the local network, and distribute to these peers the jobs while rendering the clips.

A tipical configuration is:
Node 1: 192.168.0.1 - NextFractal
Node 2: 192.168.0.2 - NextFractal
Node 3: 192.168.0.3 - JXTASpool
Node 4: 192.168.0.4 - JXTASpool
Node 5: 192.168.0.5 - JXTASpool
Node 6: 192.168.0.6 - ...
For this configuration the file seeds.txt is empty (for every node).

An alternative configuration is:
Node 1: 192.168.0.1 - NextFractal
Node 2: 192.168.0.2 - NextFractal
Node 3: 192.168.0.3 - JXTARendezvous
Node 4: 192.168.0.4 - JXTASpool
Node 5: 192.168.0.5 - JXTASpool
Node 6: 192.168.0.6 - ...
For this configuration the file seeds.txt contains only one seed (the rendezvous node):
tcp://192.168.0.3:9702

How to build the application

Download Eclipse SDK 3.5 or later from http://www.eclipse.org/downloads. Download the source code of NextFractal from the mercurial repository or download the latest snapshot from file releases. Unzip and run Eclipse, then import the main bundles ("Import existing projects on workspace"). Open the file NextFractal.product which is located in bundle com.nextbreakpoint.nextfractal.product. Run the export wizard ("Eclipse product export wizard") and select the destination folder where you want to package the application. The wizard will create the executable and the folders features, plugins, and configuration. The generated launcher for Mac OS X has a bug which requires to replace the executable NextFractal.app/Contents/MacOS/NextFractal and the related library eclipse_<version>.so in bundle plugins/org.eclipse.equinox.launcher.cocoa.macosx.<arch>_<bundle-version> with the precompiled files which are located in folder patches/macosx (copy the files for your architecture).
